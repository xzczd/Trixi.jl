Trixi.Trixi — Module
Trixi

Trixi.jl is a numerical simulation framework for hyperbolic conservation laws. A key objective for the framework is to be useful to both scientists and students. Therefore, next to having an extensible design with a fast implementation, Trixi is focused on being easy to use for new or inexperienced users, including the installation and postprocessing procedures.

See also: trixi-framework/Trixi.jl

Calculate error norms and other analysis quantities to analyze the solution during a simulation, and return the L2 and Linf errors. dg and mesh are the DG and the mesh instance, respectively. time, dt, and step refer to the current simulation time, the last time step size, and the current time step count. Trixi.convtest — Method
convtest(parameters_file, iterations; parameters...)

Run multiple Trixi simulations with the parameters in parameters_file and compute the experimental order of convergence (EOC) in the $L^2$ and $L^\infty$ norm. The number of runs is specified by iterations and in each run the initial refinement level will be increased by 1. Parameters can be overriden by specifying them as additional keyword arguments, which are passed to the respective call to run..

Trixi.integrate — Method
integrate(func, u, dg::Dg2D; normalize=true)
integrate(u, dg::Dg2D; normalize=true)

Call function func for each DG node and integrate the result over the computational domain.

The function func is called as func(u_local) for each volume node (i, j) and each element_id, where u_local is an SVectorized copy of u[:, i, j, element_id]. If normalize is true, the result is divided by the total volume of the computational domain. If func is omitted, it defaults to identity.

Examples

Calculate the integral over all conservative variables:

state_integrals = integrate(dg.elements.u, dg)

Trixi.integrate — Method
integrate(func, dg::Dg2D, args...; normalize=true)

Call function func for each DG node and integrate the result over the computational domain.

The function func is called as func(i, j, element_id, dg, args...) for each volume node (i, j) and each element_id. Additional positional arguments args... are passed along as well. If normalize is true, the result is divided by the total volume of the computational domain.

Examples

Calculate the integral of the time derivative of the entropy, i.e., ∫(∂S/∂t)dΩ = ∫(∂S/∂u ⋅ ∂u/∂t)dΩ:

# Calculate integral of entropy time derivative
dsdu_ut = integrate(dg, dg.elements.u, dg.elements.u_t) do i, j, element_id, dg, u, u_t
  u_node   = get_node_vars(u,   dg, i, j, element_id)
  u_t_node = get_node_vars(u_t, dg, i, j, element_id)
  dot(cons2entropy(u_node, equations(dg)), u_t_node)
end

Trixi.run — Method
run(parameters_file; verbose=false, refinement_level_increment=0, parameters...)

Run a Trixi simulation with the parameters in parameters_file. Parameters can be overriden by specifying them as keyword arguments (see examples).

If verbose is true, additional output will be generated on the terminal that may help with debugging. If a value for refinement_level_increment is given, initial_refinement_level will be increased by this value before running the simulation (mostly used by EOC analysis).

Examples

julia> Trixi.run("examples/parameters.toml", verbose=true)
[...]

Without changing the parameters file we can start a simulation with polydeg = 1 and t_end = 0.5 as follows:

julia> Trixi.run("examples/parameters.toml", polydeg=1, t_end=0.5)
[...]
